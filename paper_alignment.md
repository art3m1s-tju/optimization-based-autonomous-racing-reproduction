# 论文对齐说明

本文档说明当前工程复现与论文《Optimization-Based Autonomous Racing of 1:43 Scale RC Cars》的对应关系，并刻意区分“算法结构一致”和“实现细节完全相同”这两件事。

## 1. 当前已经对齐的核心结构

- 动力学模型：使用 6 维动态自行车模型，MPCC 额外引入路径进度状态 `theta` 和进度变量 `v`。
- MPCC 结构：采用中心线参数化、轮廓误差与滞后误差、进度奖励、赛道软约束、LTV 线性化和 RTI 风格热启动。
- HRHC 结构：保留“高层 trim 搜索 + 低层跟踪优化”的双层结构。
- 赛道约束：采用赛道中心线局部切向生成的平行半空间近似，并通过松弛变量软化。
- 在线求解流程：每个采样时刻基于上一次解平移得到线性化点，再求解一次局部优化问题。

## 2. 与论文实现不完全相同、但属于工程等价近似的部分

- 论文使用 FORCES 生成的定制内点 QP 求解器；当前代码使用 CasADi + `qpoases`。
- 论文强调嵌入式平台 50 Hz 实时运行；当前 Python 原型主要验证控制结构和闭环行为，不代表嵌入式部署时延。
- 论文对问题数据做了专门缩放与终止准则调优；当前实现保留了原始量纲下的直接建模。
- 论文描述的是更系统的多阶段 QP 结构与软约束范数处理；当前代码采用了与其目标一致的简化标量松弛实现。
- 当前 HRHC 的高层轨迹库是工程化简版，保留了“整段时域选一个 trim”的思想，但没有完整复刻文中所有库构造与避障走廊模块。

## 3. 工程迭代时不要轻易改动的部分

- `mpcc_controller.py` 中 `q_c / q_l / gamma` 的语义。
- 进度状态 `theta` 与进度输入 `v` 的积分关系。
- 基于上一轮解移位的热启动逻辑。
- 赛道边界软约束的结构。
- HRHC 中“全时域单一 trim”的规划假设。

## 4. 如果目标是继续向论文结果逼近

- 优先替换求解器与缩放策略，而不是先改代价函数结构。
- 优先补高层走廊/避障生成模块，而不是先增加更多启发式修正。
- 优先做批量回归与圈速、进度、求解耗时统计，再决定是否调整权重。
- 如需对齐论文实时性，应迁移到代码生成和更贴近论文的结构化 QP 求解器路线。

## 5. 当前仓库里已经为这条路线铺好的接口

- `mpcc_controller.py` 中的 `build_stage_qp_data(...)`：导出逐阶段线性化数据，便于接结构化 QP 求解器。
- `mpcc_codegen.py`：导出动力学、离散模型、Jacobian、赛道几何和误差函数的 C 代码。
- `mpcc_metadata.json`：导出求解器替换时需要的维度、权重和约束元数据。
